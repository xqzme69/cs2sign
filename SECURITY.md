# Security Policy

## Supported Versions

Security fixes are handled on the `main` branch.

## Reporting A Vulnerability

If you find a vulnerability in `cs2sign`, report it privately through GitHub
private vulnerability reporting when available. If private reporting is not
available, open an issue with a minimal public description and avoid posting
weaponized exploit details.

Useful reports include:

- Affected commit or release.
- Windows version and build environment.
- Steps to reproduce.
- Expected and actual behavior.
- Why the issue is security relevant.

Out-of-scope reports:

- Requests for injection, hooks, memory writes, anti-cheat bypasses, or gameplay
  automation.
- Game update breakage where signatures simply need refreshing.
- Missing optional signatures without a security impact.

## Disclosure

Maintainers will acknowledge valid reports, investigate, and publish a fix or
mitigation when appropriate. Public disclosure should wait until a fix is
available or maintainers agree it is safe to disclose.
